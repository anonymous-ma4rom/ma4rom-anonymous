## 三、具体压页方案

### 1. References：优先处理，预计节省最多

当前多数参考文献同时输出 DOI 和相同 DOI URL，例如：

```text
https://doi.org/10.xxxx/xxxx
https://doi.org/10.xxxx/xxxx
```

修改规则：

1. 文献有 `doi` 字段时，删除重复的 `url` 字段。
2. 正式出版文献不再保留出版社网页 URL。
3. W3C 标准、OAEI 页面和只有网页版本的文献保留 URL。
4. ResearchGate 链接只在没有 DOI、正式会议页面或 arXiv 页面时保留。
5. 删除 `preprint/online version`、`main Track`、`first published online in 2022` 等非必要附加说明。

优先检查 References 1、4、5、8、10–14、16、18、20–23、28、29。这些条目存在 DOI 与 URL 重复或冗长链接。

### 2. Introduction：删除 Related Work 已经说明的内容

第 1–2 页 Introduction 中以下内容与 Related Work 重复：

- BootOX 等 rule-based/schema matching 方法的详细介绍；
- LLM4VKG 的背景介绍；
- Geng et al. 的 SemTab Agent 介绍。

Introduction 只需要保留：

1. R2O mapping 的作用；
2. 三个具体挑战；
3. MA4ROM 的总体解决方案；
4. 三项贡献。

可以删除或压缩为一句的内容：

```text
To alleviate this burden, prior research has explored automatic or
semi-automatic approaches ...
```

到：

```text
Recent work has also explored LLM-based agents ...
```

这些文献仍保留在 Related Work，不会丢失引用。

### 3. Method 开头和 4.1 只保留一份框架介绍

Section 4 开头两段与 4.1 第一段重复表达：

```text
We present MA4ROM ...

MA4ROM coordinates four specialized agents ...
```

建议 Section 4 开头只保留一句：


### 4. Section 4.1：每个 Agent 只保留功能，不提前重复算法

4.1 的目标是介绍框架，不是重复 4.2 和 4.3。

每个 Agent 保留一至两句：

- Pattern Recognition：规则与 LLM 判断，冲突时使用 instance values 重新判断。
- Class and Datatype Property Mapping：解决 DP 与 type-column subclass mapping，细节指向 4.2。
- Object Property Mapping：完成缺失 FK、类精化与 OP 选择，细节指向 4.3。
- R2RML Mapping Generation：生成 R2RML mapping document。

删除 4.1 中对 candidate generation、domain/range、context enhancement 的细节展开。

### 5. Section 4.2：算法、公式和例子各承担不同内容

当前 Overall Algorithm、Algorithm 1、4.2.2 和 4.2.3 多次重复相同流程。

保留原则：

- Algorithm 1：完整流程。
- 4.2.1：只讲总体思想和 VALUE/TYPE 例子。
