# MA4ROM 全文术语与 PDF 原文核验

## 判定标准

- **原文术语**：相关论文或标准直接使用该短语，可以作为领域术语。
- **规范组合**：组成词均为领域术语，但完整短语是本文的方法或阶段命名。
- **必须修改**：当前说法存在语法、语义或与实现不一致的问题。

## 当前 PDF 中还需要统一的表述

### 1. Confidence 相关

当前同时出现：

- confidence estimation
- confidence judgement
- high-confidence mapping
- low-confidence mapping
- Confidence（实验设置）

建议统一为：

- 小节标题：**Candidate Ranking and Score-Margin Confidence Assessment**
- 方法描述：**maximum-score and score-margin confidence assessment**
- 门控机制：**confidence-controlled context enhancement**
- 实验设置：**Confidence-controlled**

其中
\[
c_{\max}(L)=s_1,\qquad c_{\mathrm{margin}}(L)=s_1-s_2.
\]

这两个量是排名置信信号，不应称为 calibrated probabilities。

### 2. Context 相关

当前将连接后的数据称为 schema information，但
\[
E(X)=X\Join R_1\Join\cdots\Join R_m
\]
包含真实实例元组，而不只是 schema。

建议统一写：

> context enhancement using instance values and relational context from joined tables

如果同时强调连接结构，可写：

> instance values and foreign-key-based relational context

### 3. TYPE 与 enumerated column

RODI 的原文描述是：

- a numeric type column for disjoint subclasses
- the value in a type column

因此 Figure 1、任务定义、算法、贡献和结论统一为：

- type column
- type-column values
- mapping values in a type column to ontology subclasses

`IsEnumerated(a)` 可以保留为算法内部谓词名称，但正文无需据此创造 `enumerated column` 术语。

### 4. Relationship table

正文目前混用 relation table 和 relation structure。建议使用：

- relationship table：R1 这类连接两个实体表的表
- foreign key column：R2 这类嵌入实体表中的外键列
- relationship-bearing structure：确实需要统称两者时使用一次并明确定义

避免：

- insufficient information relation structures
- relation table names
- no-information relation structures

### 5. Class hierarchy 与 class refinement

建议统一为：

- class hierarchy of the ontology
- subclass axioms
- class refinement based on subclass axioms
- refined class candidate set

不要混用 ontology hierarchy、ontology class hierarchy、class closure for endpoint refinement。

### 6. Logic rules

当前规则表中的 R1：
\[
C^{*}=\{C\}\cup Ancestor(C)\cup Descendants(C)
\]
属于集合定义，不是形式为前件推出后件的一阶逻辑规则。因此有两个选择：

1. 全文改成 **ontology-based rules**；
2. 将 R1 重写为严格的推导规则后保留 **logic rules**。

从当前论文内容看，建议采用：

> object property mapping using ontology-based rules

## PDF 原文链接

1. **RODI 2015: A Benchmark for Automatic Mapping Generation in Relational-to-Ontology Data Integration**  
   PDF: https://link.springer.com/content/pdf/10.1007/978-3-319-18818-8_2.pdf  
   支持：relational-to-ontology、type column、class hierarchy、relationship tables。

2. **RODI: Benchmarking Relational-to-Ontology Mapping Generation Quality**  
   PDF: https://journals.sagepub.com/doi/pdf/10.3233/SW-170268  
   支持：Missing FKs、foreign key constraints、class hierarchy、datatype/object property matching。

3. **Conceptually-grounded Mapping Patterns for Virtual Knowledge Graphs**  
   PDF: https://arxiv.org/pdf/2012.01917  
   支持：ontology axioms、object/data property names、foreign key constraints、inclusion dependencies。

4. **A Survey of Approaches to Automatic Schema Matching**  
   PDF: https://link.springer.com/content/pdf/10.1007/s007780100057.pdf  
   支持：schema-level information、instance-level information，以及二者的区别。

5. **Useful Confidence Measures: Beyond the Max Score**  
   PDF: https://arxiv.org/pdf/2210.14070  
   支持：maximum-score confidence 和 top-two margin confidence。

6. **Discovering Approximate Inclusion Dependencies**  
   PDF: https://www.vldb.org/pvldb/vol18/p1210-su.pdf  
   支持：approximate inclusion dependency discovery。

7. **R2RML: RDB to RDF Mapping Language**  
   W3C 官方规范：https://www.w3.org/TR/r2rml/  
   学术论文 PDF：https://arxiv.org/pdf/1804.01405  
   支持：R2RML mapping document。W3C Recommendation 没有稳定的官方 PDF 版本，不能伪造 PDF 地址。
